# Contributing

Thanks for considering a contribution.

This project is intentionally small and focused, so the best contributions are usually simple, practical, and easy to review.

## The easiest ways to help

- open an issue for a bug
- suggest a UX improvement
- propose a new session-state workflow
- improve documentation
- send a pull request with a focused fix

## Before opening a pull request

Please try to keep changes:

- minimal
- readable
- consistent with the current plugin architecture
- specific to one problem at a time

This project prefers small, high-confidence improvements over large rewrites.

## Good pull requests

A strong PR usually includes:

- a clear problem statement
- a minimal implementation
- a short explanation of the tradeoff
- updated docs if behavior changes

## Reporting bugs

When possible, include:

- what you expected to happen
- what actually happened
- your OpenCode version
- your terminal environment
- your relevant `tui.json` plugin config
- steps to reproduce the issue

## Feature requests

Feature requests are welcome, especially if they improve:

- session-state clarity
- plugin reliability
- theme-mapping UX
- multi-session usability
- maintainability

## Questions

If you are unsure whether something belongs in an issue or a PR, open an issue first.

Thanks again. Any help with issues, bug fixes, docs, or new ideas genuinely helps move the project forward.
